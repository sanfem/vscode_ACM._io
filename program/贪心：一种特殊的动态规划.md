贪心是一种堆每一个状态都进行当前最优考虑的算法，与动态规划有些区别，贪心并不怎么考虑对后面状态的影响。
贪心的难点不在于程序的设计而在于起正确性证明：
 可以用贪心算法解决的题目需要满足以下性质：

- [最优子结构]：一个问题的最优解包含其子问题的最优解
- [贪心选择性]：所求问题的整体最优解可以通过一系列局部最优的选择来到达，即通过贪心选择来达到
常用的证明方式有归纳法和交换论证法：
归纳法：
	归纳法利用了数学归纳法的思想来证明，即证明在n的条件下（n为范围内任意自然数，比如算法步数或是问题规模）命题都成立。首先证明在n为初始条件下时命题成立，在证明在n为任意状态+1时同样成立，因为在这两个条件成立时可以推导出任意条件下命题都成立。
```
F(1)=true，F(n)=true;
则F(n+1)=true
```
交换法：
	在使用简单方法得出最优答案与用贪心得到的答案进行比对
